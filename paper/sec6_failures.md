## 6.1  Reframe — bucket matrix, not case studies

§6 in v2 was framed as two case studies (LC_VCO and the
cobi_matching combinatorial workload). v3 drops the cobi_matching
case study entirely per the user D1-evening pivot and reframes §6
as a single **11 LLM × 1 failure-probe spec bucket matrix**: a
deliberately-near-saturation variant of the LC_VCO_20G spec
contract that surfaces failure modes comparably across vendors.
At ≥5 seeds per LLM the §6 budget is **55+ runs** (mlcad_runner
D8 deliverable, separate from the §5 success-Pareto runs).

## 6.2  Failure buckets

We classify each failed run into exactly one of eight buckets,
ordered roughly by the layer at which the failure surfaces:

1. **Empty-diff loop (stuck).** The LLM proposes a design vector
   identical to the previous iteration (3-iter stuck-streak); the
   iteration guard aborts (§4.3 termination (ii)).
2. **Iteration-budget exhausted (wandering).** Design vectors keep
   changing but no all-metric-pass is reached before MAX_ITER (§4.3
   termination (iii)). Distinct diagnostic signal from #1: stuck
   points at LLM-determinism, wandering at search-strategy.
3. **Contract-violation repair termination.** The LLM emits a tool
   call that violates the JSON contract; one repair-prompt retry
   fails; the run aborts.
4. **Dump-status `UNMEASURABLE` saturation.** The spec evaluator
   declares one or more metrics infeasible for several iterations
   running (no recovery path).
5. **Topology-induced sanity-range FAIL.** The LLM converges to a
   design vector outside the topology's physically reasonable
   range (e.g. tank inductance below process minimum).
6. **PDK-content refusal (red-team probe).** The LLM emits a tool
   call whose argument explicitly requests foundry content (e.g.
   `read_circuit(lib="<foundry_pdk>", cell="<nmos_model>")`);
   blocked by §3 scrub layers but counted as a failure event for
   the matrix.
7. **Reasoning-content replay** *(post-patch-era bucket; pre-patch
   this would have been an outright reasoning-content leak rather
   than a re-entry anomaly — cross-link to §3.3 / §3.5 Tier 3)*.
   The LLM paraphrases PDK-tainted content from a prior tool result
   inside its thinking channel; the redacted re-entry path means no
   foundry tokens reach history but the iteration is counted as
   anomalous.
8. **Rate-limit / 429 cascade.** Provider rate-limits trip an
   exponential-back-off chain that effectively halts iteration
   before the budget cap.

## 6.4  Pre-registered hypotheses

We **pre-register** the following predictions ahead of D8 data
collection. Table 4 will be updated post-D8 with per-bucket
✓ (confirmed) or ✗ (refuted) markers, independently of the SP3
Pareto findings in §5. Pre-registration is intentional: it converts
the temporal asymmetry (skeleton before data) from a defect into a
methodological strength.

- **H1.** Cost-floor checkpoints (Haiku 4.5, GPT-5.4-mini, DeepSeek
  V4-flash, MiMo v2.5-pro) will be over-represented in the
  *rate-limit / 429 cascade* bucket relative to frontier
  checkpoints.
- **H2.** Frontier checkpoints (Opus 4.7, GPT-5.5, Gemini 2.x) will
  be over-represented in the *`UNMEASURABLE` saturation* bucket
  (they push deeper into infeasible regions before giving up).
- **H3.** *Reasoning-content replay* (closed by the e750189c patch
  on the re-entry leg) will be the rarest bucket on patched
  checkpoints; whether DeepSeek and MiMo behave equivalently is the
  open empirical question their cells in Table 4 will answer.
