### 关键差距分析

**本地模型的优势**：
- PDK 数据不出本机，避免 NDA 风险
- 无 API 调用延迟和限速
- 可针对特定工艺节点微调
- 不受云端服务可用性影响

**本地模型的劣势**：
- 14B 模型在复杂多步推理上与 Gemini 2.5 Flash 有明显差距
- 对电路设计领域的预训练知识较少
- 上下文窗口短，难以一次处理大量仿真数据

**实用建议**：本地模型适合作为**第一层筛选和快速迭代**（调参建议、格式化输出），复杂的架构决策和深度分析可回退到 Gemini 2.5 Flash。

## 5. 部署建议

### 快速开始

```bash
# 安装首选模型
ollama pull qwen3:14b

# 测试运行
ollama run qwen3:14b "你是一个模拟电路设计助手。请分析一个两级运放的增益带宽积与相位裕度的权衡关系。"

# 作为 API 服务（供 virtuoso-agent 调用）
# Ollama 默认在 http://localhost:11434 提供 OpenAI 兼容 API
curl http://localhost:11434/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model": "qwen3:14b", "messages": [{"role": "user", "content": "test"}]}'
```

### System Prompt 模板

```
你是一名资深模拟IC设计工程师助手。你的任务是根据仿真结果分析电路性能，并以结构化 JSON 格式输出调整建议。

输出格式：
{
  "analysis": "对当前仿真结果的分析",
  "issues": ["发现的问题列表"],
  "suggestions": [
    {
      "parameter": "需调整的参数（如 M1 的 W/L）",
      "current_value": "当前值",
      "suggested_value": "建议值",
      "expected_impact": "预期影响",
      "confidence": "high/medium/low"
    }
  ]
}
```

### 多模型策略

在 virtuoso-agent 中可以实现智能路由：
- **快速任务**（格式化、参数查询）→ Qwen3-14B（快）
- **深度推理**（权衡分析、调试建议）→ DeepSeek-R1-14B（准）
- **需要 Gemini 级别能力时** → 回退到云端 Gemini 2.5 Flash

## 6. 量化知识速查

| 量化格式 | 精度 | 大小比例 (vs FP16) | 质量损失 | 推荐场景 |
|----------|------|---------------------|----------|----------|
| FP16 | 16-bit | 100% | 无 | 有足够 VRAM 时 |
| Q8_0 | 8-bit | ~50% | 极小 | 24GB+ VRAM |
| Q5_K_M | 5-bit | ~35% | 很小 | 8B 以下模型 |
| **Q4_K_M** | **4-bit** | **~28%** | **<1% (STEM)** | **16GB VRAM 首选** |
| Q3_K_M | 3-bit | ~22% | 明显 | VRAM 极度紧张时 |

> Q4_K_M 是消费级 GPU 的黄金标准：体积缩减约 72%，STEM 任务精度损失 <1%。
